## 4.1 データセット

2024 年時点の OEIS スナップショットから生成した標準 split（\texttt{std}）を使用する。
コーパスは数列レベルで重複のない train / validation / test に分割される：

| 分割       | 数列数    |
|------------|----------:|
| 学習       | 219,765   |
| 検証       |  27,470   |
| テスト     |  27,470   |

各サンプルは最大 $L = 128$ 要素の数列プレフィックスに対応する。
数列には OEIS キーワードタグ（\texttt{easy}・\texttt{hard}・\texttt{nonn}・\texttt{sign}・\texttt{core} など）が付与されており、第 5 節の層別解析に用いる。

**std split** は OEIS の A 番号の決定論的ハッシュ値に基づいて数列を train/val/test に割り当てる。
これにより同一系統の数列が分割をまたいでリークしないことが保証される。

スケール別評価のため、
$
v =
\begin{cases}
0 & (x=0), \\
1 + \log_{10}(|x|) & (x \neq 0)
\end{cases}
$
に基づく 5 つの Magnitude バケットを定義する：

| バケット     | $v$ の範囲          | 元の値の目安                       |
|--------------|--------------------|------------------------------------|
| Small        | $v < 2$            | $|x| < 10$                         |
| Medium       | $2 \leq v < 4$     | $10 \leq |x| < 1{,}000$            |
| Large        | $4 \leq v < 7$     | $10^3 \leq |x| < 10^6$             |
| Huge         | $7 \leq v < 11$    | $10^6 \leq |x| < 10^{10}$          |
| Astronomical | $v \geq 11$        | $|x| \geq 10^{10}$                 |

## 4.2 学習設定

全モデルで共通のハイパーパラメータを用いる：

| ハイパーパラメータ   | 値                                       |
|--------------------|------------------------------------------|
| エポック数          | 200（早期停止なし）                       |
| バッチサイズ        | 32（勾配累積 2 ステップ、実効 64）         |
| 学習率              | $5 \times 10^{-5}$                       |
| ウォームアップ比率  | 10%                                      |
| オプティマイザ      | AdamW、weight decay $0.01$               |
| 数値精度            | FP32（AMP 無効）                         |
| マスク確率          | 0.15                                     |
| 乱数シード          | 42                                       |
| フレームワーク      | PyTorch 2.9.1、CUDA 12.8                |
| GPU                 | GeForce RTX 3070 Ti（VRAM 8 GB）× 1     |
| CPU                 | AMD Ryzen 9 5900X 12-Core               |
| メインメモリ        | 128 GB                                  |

OEIS の値は $\log_{10}(|x|) \approx 210$ に達するものが存在するため、FP16 では桁あふれが発生する。
このため全計算に FP32 を強制し、AMP（Automatic Mixed Precision）を無効化する。

**学習時間。** ログより実測した 1 エポックあたりの学習時間（訓練フェーズ + 検証フェーズ合計）は、Small 約 5 分、Middle 約 10 分、Large 約 24 分であった。全 200 エポックの総学習時間は Large で約 80 時間に相当する。

**メモリ制約について。** VRAM 8 GB というコンシューマー GPU の制約の下、バッチサイズ 32・勾配累積 2 ステップ（実効バッチサイズ 64）に設定した。
Large モデル（\textasciitilde110M パラメータ）の FP32 学習は特にメモリを消費するため、より大規模なハードウェア環境では実効バッチサイズの拡大や追加の大規模実験が可能である。

モデル選択には全エポック中で最も低い検証損失を示すチェックポイントを用いる。
最終評価は保留テスト split に対して最終エポックのチェックポイントを使用して実施する。

## 4.3 評価指標

**Magnitude 精度（Acc$_{0.5}$）**: マスク位置のうち $|\hat{v}_i - v_i| < 0.5$ を満たす割合。元の整数スケールでは $\sqrt{10} \approx 3.16$ 倍の許容誤差に相当する。

**符号精度（Sign Accuracy）**: マスク位置のうち予測符号クラスが正解と一致する割合。

**平均 Modulo 精度（MMA：Mean Modulo Accuracy）**: 100 個の法すべてに対する分類精度の平均（マスク位置のみ）。

**Solver Top-$k$ 精度**: Solver モジュールが生成した上位 $k$ 件の候補のうち、真の次項と完全一致するものの割合（第 5.4 節）。

**正規化情報利得（NIG：Normalised Information Gain）**: 法 $m$ に対して $\text{NIG} = 1 - \mathcal{L}_{\text{CE}}^{(m)} / \log m$ で定義される。一様事前分布と比較して $x \bmod m$ についてどれだけの情報を獲得しているかを測る。NIG が高いほど、そのモジュラスに対して強い周期構造が学習されていることを示す。
