# `src/intseq_bert/train.py` Implementation Specification

## 1. Overview

This module is the pre-training entry point for IntSeqBERT, the Vanilla Transformer, and the Ablation model. It loads dataset splits and feature files generated by `preprocess.py`, runs the masked sequence modeling task, and combines magnitude, sign, and modulo losses with fixed weights:

```text
w_mag = 1.0, w_sign = 1.0, w_mod = 2.0
```

Validation metrics are reported as percentages whenever they represent accuracy.

## 4. Label Preparation

The collator output and model label format differ, so the training loop converts labels before the forward pass.

Collator output:

| Key | Shape | Content |
|-----|-------|---------|
| `mag_labels` | `(B, L, 4)` | `[log_val, sign+, sign-, sign0]` |
| `mod_labels` | `(B, L, 100)` | Integer residues; unmasked positions use `IGNORE_INDEX` |
| `mask_matrix` | `(B, L)` | Boolean mask positions |

Model label format:

| Key | Shape | Content |
|-----|-------|---------|
| `mag_targets` | `(B, L)` | `log_val` only |
| `sign_targets` | `(B, L)` | Class index: 0=Pos, 1=Neg, 2=Zero |
| `mod_targets` | `(B, L, 100)` | Integer residues |
| `mask_map` | `(B, L)` | Boolean mask positions |

Conversion:

```python
def prepare_labels(batch: Dict, device: torch.device) -> Dict:
    mag_features = batch["mag_inputs"].to(device)
    mod_features = batch["mod_inputs"].to(device)
    token_ids = batch["token_ids"].to(device)
    token_labels = batch["token_labels"].to(device)
    src_key_padding_mask = (batch["attention_mask"] == 0).to(device)

    mag_labels = batch["mag_labels"].to(device)
    mag_targets = mag_labels[:, :, 0]
    sign_targets = mag_labels[:, :, 1:4].argmax(dim=-1)
    mod_targets = batch["mod_labels"].to(device)
    mask_map = batch["mask_matrix"].to(device)
    return {
        "mag_features": mag_features,
        "mod_features": mod_features,
        "token_ids": token_ids,
        "token_labels": token_labels,
        "src_key_padding_mask": src_key_padding_mask,
        "labels": {
            "mag_targets": mag_targets,
            "sign_targets": sign_targets,
            "mod_targets": mod_targets,
            "token_targets": token_labels,
            "mask_map": mask_map,
        },
    }
```

## 5. Functions

### 5.1 `evaluate()`

Runs inference over the validation or test dataset and computes metrics.

Metrics:

1. **Sign accuracy (%)**
   - Classification accuracy for Positive, Negative, and Zero.
   - Formula: `(pred_class == target_class)[mask_map].mean() * 100`.

2. **Mean modulo accuracy (%)**
   - Computes classification accuracy for each of the 100 moduli and averages them.
   - Formula: `mean((mod_pred == mod_target)[mask_map]) * 100`.

3. **Magnitude accuracy (%)**
   - Regression converted to tolerance accuracy.
   - Correct if `abs(pred - target) < 0.5`.
   - On the log10 scale, a 0.5 error corresponds to about a `sqrt(10) ~= 3.16x` multiplicative range.

4. **Magnitude MSE**
   - Reference regression metric where lower is better.

Return value:

```python
{
    "val_loss": float,
    "sign_acc": float,
    "mod_acc": float,
    "mag_acc": float,
    "mag_mse": float,
}
```

### 5.2 `train()`

Setup:

- Set random seed.
- Load train/validation datasets.
- Create `OEISCollator` and `DataLoader`.
- Initialize model and move it to device.
- Create `AdamW` optimizer and scheduler.
- Initialize early stopping.

Mixed precision is intentionally disabled. OEIS data contains values around `10^210`, which can produce unstable intermediate values under FP16. Training therefore uses FP32 instead of `autocast` and `GradScaler`.

Epoch flow:

1. **Training phase**
   - Set `model.train()`.
   - Convert labels with `prepare_labels()`.
   - Run forward, loss, backward.
   - Step optimizer every `accum_steps`.
   - Log total loss and validation metrics.

2. **Validation phase**
   - Set `model.eval()`.
   - Call `evaluate()`.
   - Print a summary such as:

```text
Epoch 1: Loss=2.5, Mag Acc=85.2%, Sign Acc=98.1%, Mod Acc=45.3%
```

3. **Checkpointing**
   - Save `best_model.pt` when `val_loss` improves.
   - Always update `last_checkpoint.pt`.

4. **Early stopping**
   - Stop when `val_loss` does not improve for `patience` epochs.

## 7. Implementation Sketch

```python
batch = next(dataloader)
inputs = prepare_labels(batch, device)

outputs = model(
    inputs["mag_features"],
    inputs["mod_features"],
    src_key_padding_mask=inputs["src_key_padding_mask"],
    labels=inputs["labels"],
)
loss = outputs["loss"]

labels = inputs["labels"]
mask_map = labels["mask_map"]

mag_preds = outputs["predictions"]["mag_mu"]
mag_targets = labels["mag_targets"]
mag_acc = ((torch.abs(mag_preds - mag_targets) < 0.5) & mask_map).sum().float()
mag_acc = mag_acc / mask_map.sum() * 100

sign_logits = outputs["predictions"]["sign_logits"]
sign_preds = sign_logits.argmax(dim=-1)
sign_acc = ((sign_preds == labels["sign_targets"]) & mask_map).sum().float()
sign_acc = sign_acc / mask_map.sum() * 100

mod_logits = outputs["predictions"]["mod_logits"]
mod_logits_split = model._split_mod_logits(mod_logits)
mod_preds = torch.stack([l.argmax(dim=-1) for l in mod_logits_split], dim=-1)
mod_correct = (mod_preds == labels["mod_targets"]) & mask_map.unsqueeze(-1)
mod_acc = mod_correct.sum().float() / (mask_map.sum() * config.NUM_MODULI) * 100
```

## 8. Early Stopping

| Parameter | Default | Description |
|-----------|---------|-------------|
| `patience` | 5 | Number of non-improving epochs to wait |
| `delta` | 0.0 | Minimum improvement |

Behavior:

1. Record `val_loss` at the end of each epoch.
2. Reset the counter if `val_loss < best_val_loss - delta`.
3. Otherwise increment the counter.
4. Stop when the counter reaches `patience`.

```python
class EarlyStopping:
    def __init__(self, patience: int = 5, delta: float = 0.0):
        self.patience = patience
        self.delta = delta
        self.counter = 0
        self.best_loss = float("inf")

    def __call__(self, val_loss: float) -> bool:
        if val_loss < self.best_loss - self.delta:
            self.best_loss = val_loss
            self.counter = 0
            return False
        self.counter += 1
        return self.counter >= self.patience
```

## 9. Training Log Outputs

Each training run creates or updates these files under `output_dir`.

Directory layout:

```text
checkpoints/intseq_std/
├── best_model.pt
├── last_checkpoint.pt
├── config.json
├── history.csv
├── best_metrics.json
└── train.log
```

### 9.1 `config.json`

Written once at training start. It records:

- all command-line arguments,
- important `config.py` constants,
- device information and Git hash if available,
- Python/PyTorch versions,
- dataset statistics,
- resume source if applicable.

### 9.2 `history.csv`

One row per epoch. Columns include:

| Category | Columns |
|----------|---------|
| Meta | `epoch`, `lr`, `time_sec`, `is_best`, `early_stop_counter` |
| Loss | `train_loss`, `val_loss` |
| Magnitude | `val_mag_acc`, `val_mag_mse` |
| Sign | `val_sign_acc` |
| Modulo summary | `val_mod_acc`, `val_mod_loss` |
| Modulo per modulus | `mod_acc_2` through `mod_acc_101` |
| Weights | `w_mag`, `w_sign`, `w_mod` |

Saving all per-modulus accuracies enables later analyses such as mod-7 learning curves or prime-vs-composite comparisons.

### 9.3 Representative Console Mods

To avoid printing all 100 moduli, logs show representative moduli:

| Modulus | Rationale |
|---------|-----------|
| `mod 2` | Parity |
| `mod 3` | Ternary residue |
| `mod 5` | Small prime |
| `mod 7` | Medium prime |
| `mod 10` | Base-10 bias probe |
| `mod 100` | Larger modulus ability |
| `mod 101` | Large prime |

Example:

```text
Epoch 1 Results:
  Train Loss: 0.6532
  Val Loss:   0.2717
  Mag Acc:    77.55% (MSE: 0.342)
  Sign Acc:   85.21%
  Mod Acc:    13.24% (Mean of 100 mods)
    mod2: 75.2%, mod3: 62.1%, mod5: 48.3%, mod7: 35.6%, mod10: 42.1%, mod100: 8.5%, mod101: 7.9%
```

### 9.4 Checkpoint Format

`best_model.pt` and `last_checkpoint.pt` store:

```python
{
    "epoch": int,
    "model_state_dict": OrderedDict,
    "optimizer_state_dict": dict,
    "scheduler_state_dict": dict,
    "val_loss": float,
    "val_metrics": dict,
    "config": dict,
}
```

### 9.5 Resume Behavior

| File | Behavior | Reason |
|------|----------|--------|
| `config.json` | Keep existing | Preserve original run settings |
| `history.csv` | Append | Preserve continuous history |
| `best_model.pt` | Overwrite only on improvement | Preserve best model |
| `last_checkpoint.pt` | Overwrite every epoch | Always keep latest state |
| `best_metrics.json` | Overwrite only on improvement | Sync with best model |
| `train.log` | Append | Preserve session history |

## 10. Test-Only Mode (`--test_only`)

Test-only mode evaluates an already trained checkpoint on train/val/test data without running training.

Additional CLI arguments:

| Argument | Type | Default | Description |
|----------|------|---------|-------------|
| `--test_only` | flag | `False` | Enable test-only mode |
| `--model_path` | str | `None` | Checkpoint path; required in test-only mode |
| `--test_split` | str | `"test"` | Split to evaluate: `train`, `val`, or `test` |
| `--test_output` | str | `None` | Optional CSV output path |

Validation:

```python
if args.test_only and not args.model_path:
    parser.error("--test_only requires --model_path")
if args.model_path and not args.test_only:
    parser.error("--model_path requires --test_only flag")
```

Flow:

1. Restore model configuration from the checkpoint or adjacent `config.json`.
2. Load the checkpoint's `model_state_dict`.
3. Load the requested split.
4. Run `evaluate()`.
5. Write CSV-compatible output and detailed JSON.

Skipped components:

- Optimizer and scheduler initialization,
- GradScaler initialization,
- Early stopping,
- training loop,
- checkpoint saving,
- train-log writing.

### 10.1 CSV Output

`test_results.csv` uses a `history.csv`-compatible column layout. Training-only fields receive neutral values, such as `epoch=0`, `lr=0.0`, and `train_loss=0.0`.

### 10.2 JSON Output

`test_metrics.json` records model path, split, sample count, evaluation time, aggregate metrics, representative modulo metrics, all modulo accuracies, and evaluation timestamp.

### 10.3 Model Config Restoration Priority

| Priority | Source | Description |
|----------|--------|-------------|
| 1 | `checkpoint["config"]` | Strongest source |
| 2 | Adjacent `config.json` | Same checkpoint directory |
| 3 | CLI arguments | Fallback; may cause size mismatch |

### 10.4 Shared CSV Headers

`TrainingLogger.get_csv_headers()` should define the header list used by both `history.csv` and `test_results.csv`, avoiding duplicate definitions.

## 11. Usage Examples

Evaluate on the test split:

```bash
python -m intseq_bert.train \
  --test_only \
  --model_path checkpoints/intseq_std/best_model.pt \
  --split_type std \
  --output_dir checkpoints/intseq_std/
```

Evaluate on validation data:

```bash
python -m intseq_bert.train \
  --test_only \
  --model_path checkpoints/intseq_std/best_model.pt \
  --split_type std \
  --test_split val \
  --output_dir checkpoints/intseq_std/
```

Cross-evaluate an easy-split model on the std test set:

```bash
python -m intseq_bert.train \
  --test_only \
  --model_path checkpoints/intseq_easy/best_model.pt \
  --split_type std \
  --test_split test \
  --output_dir cross_eval/easy_on_std/
```
